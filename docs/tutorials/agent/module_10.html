10.1 Production Checklist

生产检查清单
  • ✓ Rate limiting — prevent API cost overruns / 速率限制——防止API成本超支
  • ✓ Error handling — retry with exponential backoff / 错误处理——指数退避重试
  • ✓ Logging — record all LLM calls and tool executions / 日志——记录所有LLM调用和工具执行
  • ✓ Cost monitoring — track spend per user/task / 成本监控——跟踪每用户/任务花费
  • ✓ Timeouts — kill long-running tasks / 超时——终止长时间运行的任务
  • ✓ Sandboxing — never execute arbitrary code on production / 沙箱——永远不在生产环境执行任意代码
  • ✓ Human-in-the-loop — require approval for high-stakes actions / 人在回路中——高风险操作需人工审批

10.2 Error Handling Pattern

错误处理模式
import time
from openai import RateLimitError, APIError

def call_with_retry(func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return func()
        except RateLimitError:
            wait = 2 ** attempt  # 1s, 2s, 4s
            print(f'Rate limited. Retrying in {wait}s...')
            time.sleep(wait)
        except APIError as e:
            print(f'API error: {e}')
            if attempt == max_retries - 1:
                raise
    raise Exception('Max retries exceeded')
