Building the Agent Loop

构建智能体循环

4.1 The Observe → Think → Act Cycle

观察→思考→行动循环

An agent is fundamentally a loop: (1) Observe the current state — user input plus any tool results, (2) Think about what to do next using the LLM, (3) Act by calling a tool or returning a final response. This loop repeats until the task is complete.

智能体本质上是一个循环：(1) 观察当前状态——用户输入加上任何工具结果，(2) 使用LLM思考下一步做什么，(3) 通过调用工具或返回最终响应来行动。此循环重复直到任务完成。

4.2 Minimal Agent — Anthropic Implementation

最简智能体——Anthropic实现
import anthropic

client = anthropic.Anthropic()

tools = [{
    'name': 'get_weather',
    'description': 'Get current weather for a city',
    'input_schema': {
        'type': 'object',
        'properties': {
            'city': {'type': 'string', 'description': 'City name'}
        },
        'required': ['city']
    }
}]

def execute_tool(name, args):
    if name == 'get_weather':
        return f"Weather in {args['city']}: 22°C, sunny"
    return 'Unknown tool'

def run_agent(user_message, max_steps=10):
    messages = [{'role': 'user', 'content': user_message}]

    for step in range(max_steps):  # Safety limit!
        response = client.messages.create(
            model='claude-sonnet-4-20250514',
            max_tokens=1024,
            system='You are a helpful assistant with tools.',
            tools=tools,
            messages=messages
        )

        if response.stop_reason == 'tool_use':
            tool_block = next(
                b for b in response.content if b.type == 'tool_use'
            )
            result = execute_tool(tool_block.name, tool_block.input)

            messages.append({
                'role': 'assistant', 'content': response.content
            })
            messages.append({
                'role': 'user',
                'content': [{
                    'type': 'tool_result',
                    'tool_use_id': tool_block.id,
                    'content': result
                }]
            })
            continue

        # Final text response
        return next(b.text for b in response.content if b.type == 'text')

    return 'Max steps reached'

print(run_agent('What is the weather in London?'))
✅Key insight: The for step in range(max_steps) loop is the heart of every agent. The LLM decides when to stop by not requesting a tool call. / 关键洞察：for step in range(max_steps)循环是每个智能体的核心。LLM通过不请求工具调用来决定何时停止。

4.3 Safety: Preventing Runaway Agents

安全：防止失控智能体
⚠️A runaway agent with no iteration limit can consume your entire API budget in minutes. ALWAYS set a maximum step count. / 没有迭代限制的失控智能体可能在几分钟内耗尽您的整个API预算。始终设置最大步数。
  • Set max_steps=10 as a sensible default / 设置max_steps=10作为合理默认值
  • Log every step so you can debug loops / 记录每个步骤以便调试循环
  • Add cost tracking per agent run / 为每次智能体运行添加成本跟踪
  • Consider a per-run token budget / 考虑每次运行的token预算
