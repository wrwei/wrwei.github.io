Memory & Context Management

记忆与上下文管理

6.1 The Context Window Problem

上下文窗口问题

Every LLM has a finite context window (e.g., 128K tokens for GPT-4o, 200K for Claude). Long conversations eventually exceed this limit. An agent needs strategies to manage what stays in context and what gets compressed or stored elsewhere.

每个LLM都有有限的上下文窗口（例如GPT-4o的128K token，Claude的200K）。长对话最终会超出此限制。智能体需要策略来管理哪些信息保留在上下文中，哪些被压缩或存储到其他地方。

6.2 Sliding Window Memory

滑动窗口记忆
class ConversationMemory:
    def __init__(self, max_messages=50):
        self.messages = []
        self.max_messages = max_messages

    def add(self, role, content):
        self.messages.append({'role': role, 'content': content})
        if len(self.messages) > self.max_messages:
            # Keep system + most recent messages
            self.messages = (
                self.messages[:1] +
                self.messages[-self.max_messages + 1:]
            )

    def get_messages(self):
        return self.messages.copy()

6.3 Summarisation Strategy

摘要策略

For long-running agents, periodically summarise older messages using the LLM itself, then replace the originals with the summary. This compresses context while retaining key information.

对于长期运行的智能体，定期使用LLM本身总结旧消息，然后用摘要替换原始消息。这在保留关键信息的同时压缩了上下文。

async def summarise_old_messages(client, messages, keep_recent=10):
    if len(messages) <= keep_recent + 5:
        return messages  # Not enough to summarise

    old = messages[1:-keep_recent]  # Skip system, keep recent
    summary_prompt = f'Summarise this conversation concisely:\n{old}'

    summary = client.messages.create(
        model='claude-sonnet-4-20250514',
        max_tokens=500,
        messages=[{'role': 'user', 'content': summary_prompt}]
    )

    return (
        messages[:1] +  # System message
        [{'role': 'user', 'content': f'[Summary]: {summary.content[0].text}'}] +
        messages[-keep_recent:]  # Recent messages
    )

6.4 Persistent Storage

持久化存储
  • Vector databases (Pinecone, Chroma, Weaviate) — for semantic search over past interactions / 向量数据库——用于过去交互的语义搜索
  • Key-value stores (Redis) — for fast access to user preferences / 键值存储——用于快速访问用户偏好
  • SQL databases — for structured data and audit trails / SQL数据库——用于结构化数据和审计跟踪
