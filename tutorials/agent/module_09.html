Testing & Evaluation

测试与评估

9.1 Why Testing Agents is Hard

为什么智能体测试很难

Agents are non-deterministic: the same input can produce different outputs. They make multi-step decisions where errors compound. Testing requires evaluating both individual steps (did it pick the right tool?) and end-to-end outcomes (did the user get the right answer?).

智能体是非确定性的：相同的输入可能产生不同的输出。它们进行多步骤决策，错误会累积。测试需要评估单个步骤（是否选择了正确的工具？）和端到端结果（用户是否得到了正确答案？）。

9.2 Testing Strategies

测试策略
  • Unit test each tool independently — verify inputs/outputs / 独立单元测试每个工具——验证输入/输出
  • Mock the LLM for deterministic flow testing / 模拟LLM进行确定性流程测试
  • Evaluation datasets with expected outcomes / 带有预期结果的评估数据集
  • LLM-as-judge for subjective quality assessment / LLM作为评判进行主观质量评估
  • Track token usage and cost per task / 跟踪token使用量和每任务成本
  • Regression tests when changing prompts / 更改提示时进行回归测试

9.3 Simple Evaluation Framework

简单评估框架
test_cases = [
    {
        'input': 'What is 25 * 17?',
        'expected_tool': 'calculate',
        'expected_contains': '425'
    },
    {
        'input': 'Weather in Tokyo?',
        'expected_tool': 'get_weather',
        'expected_contains': 'Tokyo'
    }
]

def evaluate(agent, test_cases):
    results = []
    for tc in test_cases:
        output = agent.run(tc['input'])
        passed = tc['expected_contains'] in output
        results.append({
            'input': tc['input'],
            'output': output,
            'passed': passed
        })
    pass_rate = sum(r['passed'] for r in results) / len(results)
    print(f'Pass rate: {pass_rate:.0%}')
    return results
