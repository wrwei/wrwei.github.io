Unix Tools and Simple Automation

Learn to search, combine and process text with Unix tools, then write your first shell script.

Working with text

A large part of Unix work is processing text — reading files, searching for patterns, transforming content. The following tools are your core toolkit:

bash — text processing tools
# View file contents
cat notes.txt
# Search for a pattern
grep 
"error"
 server.log
# Case-insensitive search
grep 
-i 

 server.log
# Show line numbers
grep 
-n 

 server.log
# Count matching lines
grep 
-c 

 server.log

Pipes and redirection

The real power of Unix comes from combining small tools. A pipe (|) sends the output of one command directly into the next.

bash — pipes
# Count how many error lines are in a log file
grep 
"error" server.log 
| 
wc 
-l
47
# Find unique error messages, sorted
grep 
"error" server.log 
| 
sort 
| 
uniq
# Show the 5 most frequent errors
grep 
"error" server.log 
| 
sort 
| 
uniq 
-c 
| 
sort 
-rn 
| 
head 
-5

Redirection saves output to a file instead of printing it to the screen:

bash — redirection
# Write output to a file (overwrites)
ls > 
filelist.txt
# Append output to a file
echo 
"new entry" >> 
filelist.txt
# Redirect errors to a file
grep 
"error" server.log 
2> 
errors_only.txt
The Unix philosophy in action

Each of these commands does one thing. Pipes let you compose them into arbitrarily powerful pipelines without writing any custom code. This is the Unix philosophy: small, focused tools that can be combined freely.

Your first shell script

A shell script is simply a text file containing commands, executed in sequence. Let us write one.

Open a text editor (e.g. nano myscript.sh) and write:

bash — myscript.sh
#!/bin/bash
# This is a comment
echo 
"Hello from my first script"
echo 
"Today is: "
date

Save the file. Now make it executable and run it:

bash — run a script
chmod 
+x 
myscript.sh
./myscript.sh
Hello from my first script
Today is: Mon Jan 10 09:15:00 GMT 2025

Variables store values you can reuse:

bash — variables
#!/bin/bash
NAME=
"Alice"
echo 
"Hello, $NAME"
Hello, Alice
No spaces around =

Shell variable assignment is strict: NAME=Alice works, but NAME = Alice does not. The shell would interpret NAME as a command.

Loops

A for loop repeats a block of commands for each item in a list. This is useful when you need to process many files.

bash — for loop
#!/bin/bash
# Print the name of every .txt file
for 
file 
in 
*.txt
; do
  echo 
"Processing: $file"
  wc 
-l 
"$file"
done

To loop over files in a specific directory:

bash — loop with find
for 
file 
in 
$(find . 
-name 
"*.log"
)
; do
  echo 
"Found: $file"
done

Key terms

grep
Search for a text pattern in a file. Short for 'global regular expression print'.
pipe (|)
Connects two commands — the output of the left becomes the input of the right.
redirection (>)
Writes a command's output to a file instead of the screen.
shebang
The first line #!/bin/bash that tells the OS which interpreter to use.
variable
A named container for a value: NAME=Alice.
for loop
Repeats a block of commands for each item in a list.

Exercises

Lab exercises
  1. Download the sample log file from the lab materials. Use grep to find all lines containing the word ERROR.
  2. Pipe the result into wc -l to count how many error lines there are.
  3. Extend the pipeline to find the five most common error messages.
  4. Write a script count_files.sh that prints the name and line count of every .txt file in a given directory.
  5. Make the script executable and run it on a folder of your choice.
  6. Extension: modify the script to accept the directory path as a variable at the top of the script so it is easy to change.
